## 四、无法抓取的原因分析

| 失败原因 | 平台数 | 典型平台 |
|----------|--------|----------|
| **域名无效/DNS失败** | 1 | 广东省公共资源交易平台 (gdggzy.org.cn) |
| **严格反爬拦截** | 1 | 中国政府采购网 API (ccgp.gov.cn) |
| **SPA + 需要 JS 渲染** | 6 | 大部分平台，可用 Playwright 解决 |
| **验证码** | 3 | ctbpsp、千里马、政府采购网 GPCMS |
| **需要登录** | 1 | 千里马（部分功能） |

### 核心结论
- **requests 裸请求 90% 失败**：中国政府网站几乎都是 SPA、含验证码、或有 CDN/WAF 防护
- **Playwright 浏览器方案是必需的**：现有 GPCMS 爬虫的成功验证了这一点
- **反爬最严格**：ccgp（国家级）、千里马（商业平台）

## 五、当前项目结构

```
hermes-sports-bidding/
├── main.py                ← 统一入口（支持 --site 切换平台）
├── config.py              ← 配置文件（13个平台注册、关键词、城市）
├── storage.py             ← SQLite 存储（含 status 字段供 hermes 消费）
├── captcha_solver.py      ← 验证码识别（ddddocr）
├── crawlers/
│   ├── base.py            ← 基类（统一入库、去重hash、统计）
│   ├── gd_gpcms.py        ← 广东+福建政府采购网（GPCMS通用）
│   ├── sz_ggzy.py         ← 深圳公共资源交易中心
│   ├── ly_ggzy.py         ← 龙岩市公共资源交易中心（新增 ✅）
│   └── ccgp.py            ← 中国政府采购网（待修复反爬）
├── test_platforms.py      ← 平台联通性测试工具
├── test_data_quality.py   ← 深度数据质量测试
├── platform_test_report.json  ← 联通性测试结果
└── sports_bidding.db      ← 数据库（已有 71+ 条记录）
```

### 与 hermes 对接

- hermes 用 `SELECT * FROM bidding_records WHERE status='raw'` 获取待处理数据
- 处理完成后调用 `mark_processed(id, 'processed')`
